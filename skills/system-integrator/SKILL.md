# Elite System Integrator & Orchestrator Protocol

## When to use this skill
- **Complex Tasks:** When a request involves multiple domains (e.g., "Add a native C++ feature with Flutter UI and Python backend").
- **Strategic Refactoring:** When changing the "soul" of the system across multiple layers.
- **Crisis Management:** When a bug is deep-rooted and spans multiple skills.

## Orchestration Mandates

### 1. Strategic Delegation & User Choice
- **Role Identification:** Delegate to specialized skills.
- **The Upgrade Loop:** When a sub-skill identifies an "Elite Upgrade" (via `@qa-code-review`), present it as a choice to the user.
- **Directive Management:** 
    - If User says "Upgrade": Apply the elite standard across all relevant layers.
    - If User says "Stick to local": Enforce local consistency strictly.

### 2. Context & State Management
- **Master Map:** Maintain the "Convention Gap Report" from `@project-analyzer`.
- **Turn Compression:** Instruct all sub-tasks to use context-rich searches and parallel operations to minimize latency and token usage.

### 3. Architectural Alignment
- **Global Constraints:** Enforce `@clean-architecture-solid` across all domains.
- **Consistency Audit:** Before finishing, run a cross-domain review to ensure naming and data flows are consistent (e.g., "Does the Python DTO match the Dart model?").

## Integration Workflow
1. **Analyze:** Use `@project-analyzer` to map the entire "territory."
2. **Plan:** Create a multi-skill execution strategy.
3. **Execute:** Delegate to specialized skills in parallel or sequence.
4. **Validate:** Perform a final `@qa-code-review` across all modified layers.

## Strict Prohibition
- **NO Git Commands:** Strictly filesystem-based context.
- **Optional Testing:** Only implement tests if explicitly requested by the user.
